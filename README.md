# Specification Pattern in Delphi

A simple demon on using the Specification Pattern in Delphi to make business rules easier to read, reuse, and compose.

It uses a small Order domain example and shows how simple specifications can be combined in a service layer to express meaningful business rules cleanly.

May be of interest to Delphi developers working with business logic and looking for an alternative to long, repeated conditional code.

The demonstration uses code from Project Galahad, which is currently under development.

For more information, please see [here](https://www.beyondvelocity.com/post/222)
